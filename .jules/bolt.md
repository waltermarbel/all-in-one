## 2024-06-13 - Cached ConfigurationManager->GetConfig
**Learning:** Found a recurring pattern where `ConfigurationManager->GetConfig` continuously loads and decodes the configuration JSON file for every single settings check (e.g. `isTalkEnabled()`, `isClamavEnabled()`, `GetToken()`). This incurs a significant disk I/O and CPU penalty (JSON parsing) on repeated checks across the codebase lifecycle.
**Action:** Implemented an instance-level cache `private ?array $configCache`. Ensures reads stay fast while still keeping sync via immediate cache updates on `WriteConfig`. Next time watch for `json_decode(file_get_contents(...))` calls wrapped in getter functions called repeatedly in a loop (like `ContainerDefinitionFetcher->GetDefinition()`).

## 2024-06-22 - Cached DataConst Directory Resolution
**Learning:** In Nextcloud AIO, data and session directories (e.g., `/mnt/docker-aio-config/data/`) are statically mounted Docker volumes that do not change paths during a single request lifecycle. The `DataConst` class was previously doing `is_dir` and `realpath` filesystem checks every time paths like `GetDataDirectory()` were called. These filesystem operations can add up when used extensively, especially inside loop configurations like `ContainerDefinitionFetcher`. Caching these static directory paths is perfectly safe here, avoiding redundant disk I/O.
**Action:** Added static properties `self::$dataDirectoryCache` and `self::$sessionDirectoryCache` to cache the resolved paths inside `DataConst.php`. Note: We should NOT cache stateful statuses like Docker container status, but stateless environment variables and static mounted volume paths are great targets for in-memory caching.
